## 🚀 사용 시나리오별 가이드

### "처음 시작합니다"
1. [QUICKSTART_KR.md](QUICKSTART_KR.md) 읽기
2. 권한 설정 및 빌드
3. `./start.sh` 실행
4. 테스트

### "문제가 있어요"
1. [TESTING.md](TESTING.md)의 "문제 해결" 섹션 확인
2. GitHub Issues 검색
3. 새 이슈 생성

### "설정을 바꾸고 싶어요"
1. [docs/CONFIG.md](docs/CONFIG.md) 읽기
2. `config.json` 수정
3. 재시작

### "샘플을 추가하고 싶어요"
1. [samples/README.md](samples/README.md) 읽기
2. `samples/` 폴더에 .wav 파일 추가
3. `config.json`에서 매핑 설정

### "코드를 이해하고 싶어요"
1. [PROJECT_OVERVIEW.md](PROJECT_OVERVIEW.md) 읽기 - 전체 구조 파악
2. [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) 읽기 - 상세 아키텍처
3. 소스 코드 탐색

### "기여하고 싶어요"
1. [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) 읽기
2. Fork & 브랜치 생성
3. 코드 작성 및 테스트
4. PR 생성
